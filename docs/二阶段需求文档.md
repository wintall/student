# 二阶段需求文档：校园 AI 助手中心

版本：v1.0  
日期：2026-06-30  
状态：二阶段交付版

## 1. 项目背景

一阶段系统已经完成学生、教师、院系、班级、课程、考试、成绩、请假、课表、邮件、公告、数据体检等基础教务管理能力。

二阶段目标不是继续堆传统后台页面，而是在现有系统基础上构建一个统一的“校园 AI 助手中心”，体现 AI 产品思维、Agent 工具调用能力和多角色智能服务能力。

## 2. 产品定位

系统新增“校园助手”，以足球形态悬浮气泡作为统一入口。管理员、教师、学生都通过同一个入口使用 AI 助手。

用户可以直接自然语言提问，也可以点击输入框下方的能力组件切换助手能力。能力组件不跳转页面，只切换当前对话模式、快捷问题、工具范围和提示词上下文。

## 3. 用户角色

| 角色 | 使用目标 |
| --- | --- |
| 学生 | 查询成绩、课表、请假进度、考勤记录，获得学习辅导、情绪陪伴、知识问答 |
| 教师 | 查询授课安排、学生成绩、班级情况、请假审批、学生考勤，获得教学辅助 |
| 班主任 | 管理本班学生请假和考勤，查看本班学生情况 |
| 院系主任 | 管理本院系学生/教师相关数据，查看院系统计和异常 |
| 管理员 | 全局管理、数据体检、统计分析、权限维护、知识库管理 |

## 4. 核心需求

### 4.1 足球形态助手入口

- 右下角显示足球悬浮气泡。
- 支持点击展开问答面板。
- 支持拖拽和位置记忆。
- 展开面板显示当前角色身份。
- 保留清空会话、收起面板等基础操作。

### 4.2 统一问答面板

- 中间区域显示多轮聊天。
- 底部为输入框。
- 输入框下方显示能力组件。
- 用户可以直接输入自然语言。
- 系统根据用户问题自动识别意图并调用对应能力。

### 4.3 能力组件

| 组件 | 能力说明 |
| --- | --- |
| 自动 | 默认模式，自动识别意图 |
| RAG 知识问答 | 文学、校园制度、知识库问答 |
| 教务工具调用 | 成绩、课表、请假、公告、考勤等系统数据查询 |
| 学习辅导 | 学习计划、复习建议、课程辅导 |
| 文档处理 | 文档摘要、重点提取、提纲生成 |
| 情绪陪伴 | 学习压力、考试焦虑、人际烦恼疏导 |
| 数据分析 | 数据体检、成绩统计、异常分析 |
| 世界杯问答 | 世界杯知识、球星、球队、规则问答 |
| AI 知识问答 | LangChain、LangGraph、FastAPI、Dify、Python、Linux、SQL 等技术知识 |
| 更多 | 后续扩展能力入口 |

### 4.4 教务工具调用

系统应支持通过自然语言调用后端安全工具：

- 查询我的成绩。
- 查询我的课表。
- 查询我的请假申请。
- 查询我的考勤记录。
- 查询最近公告。
- 查询待审批请假。
- 查询班级学生考勤。
- 查询数据体检摘要。

### 4.5 请假考勤模块

现有“请假管理”建议升级为“请假考勤”模块。

模块下包含：

- 我的请假。
- 请假审批。
- 我的考勤。
- 学生考勤管理。
- 教职工考勤管理。

权限原则：

- 学生可以查看自己的请假和考勤。
- 教职工可以查看自己的请假和考勤。
- 班主任可以管理本班学生考勤。
- 院系主任可以管理本院系学生考勤，建议也可管理本院系教职工考勤。
- 管理员可以管理全部考勤。

### 4.6 情绪陪伴边界

- 可以做考试压力、学习挫败、人际烦恼的安抚和建议。
- 不做医疗诊断。
- 不承诺治疗效果。
- 识别严重风险时，引导联系老师、家长、心理老师或专业机构。

## 5. 非功能需求

- 所有接口统一返回 `{code, message, data}`。
- 工具调用必须经过权限校验。
- 不允许大模型直接操作数据库。
- 工具调用、会话消息、用户反馈应可追踪。
- 前端交互要保持轻量，不影响原后台操作。
- AI 响应慢时要有加载态和超时提示。

## 6. 验收标准

- 足球气泡可打开统一助手面板。
- 能力组件可切换模式但不跳转页面。
- 学生能通过助手查询自己的成绩、课表、请假、考勤。
- 教师能通过助手查询自己授课相关数据。
- 班主任/院系主任/admin 能按权限查看学生考勤。
- RAG 知识问答能返回答案和来源。
- AI 知识问答、世界杯问答至少具备可演示知识库。
- 文档处理具备基础摘要能力或开发预留说明。
- 四份交付文档完整。

## 8. 二阶段最终功能范围

### 8.1 足球智能助手

- 使用动态足球作为统一入口。
- 支持自动模式和手动切换能力模式。
- 支持多轮对话、短期上下文、任务草稿和待确认动作。
- 写操作必须先确认，避免误改数据库。
- 支持前端打字式输出效果，提升对话自然度。

### 8.2 教务助手

- 查询本人信息、我的课程、我的老师、我的课表、我的成绩。
- 查询学生、教师、课程、班级、院系、教室、学期、公告等基础数据。
- 管理员可在权限范围内通过自然语言新增、修改、停用数据。
- 支持缺参追问、多轮补参、确认后执行。

### 8.3 综合知识库 RAG

- 支持创建个人/公共知识库。
- 支持文本、上传文件、本地白名单路径导入。
- 支持文档切片、向量召回、关键词/BM25 召回和来源展示。
- 支持三国演义等长文本导入后问答。
- 检索到资料但模型无返回时，提供本地兜底摘要，不把内部错误暴露给用户。

### 8.4 学习辅导与诗词鉴赏

- 支持课程概念讲解、学习计划、题目解析。
- 支持诗词赏析，避免无关 RAG 资料污染回答。
- 适合演示问题：`讲解牛顿第二定律`、`赏析一下将进酒`。

### 8.5 文档处理

- 支持上传 txt、md、pdf、图片等材料。
- 支持摘要、重点提取、英汉互译。
- 图片材料通过 OCR 提取文字后再进入助手处理。
- 可将处理后的文本沉淀进综合知识库，后续继续问答。

### 8.6 情绪陪伴

- 支持用户表达压力、焦虑、低落、人际烦恼时进行共情回应。
- 使用 CBT、正念、压力应对等心理学框架给出建议。
- 不替代心理咨询和医疗诊断。
- 出现较高风险表达时，优先提醒用户寻求现实帮助，并可向管理员/关联教师发送提醒邮件，体现人在回路。

### 8.7 路线生活

- 支持地点搜索、附近吃喝玩乐查询。
- 支持路线规划，包含起点、中途点、目的地。
- 支持公共交通和驾车方案的文字化说明，预留地图可视化接入。

### 8.8 搜索引擎与专题问答

- 搜索引擎模式支持联网检索实时问题。
- 世界杯问答对实时射手榜、赛程等问题优先使用搜索结果，避免模型旧知识。
- AI 知识问答支持 LangChain、LangGraph、FastAPI、Dify、Python、Linux、SQL 等内容。

### 8.9 编程助手与 GitHub 助手

- 编程助手支持代码问答、代码生成、文件定位、代码解释，并可输入本地项目路径分析项目结构、技术栈、模块职责、风险和优化方向。
- GitHub 助手支持仓库信息、issue、PR 等自然语言操作，需配置 GitHub Token。

## 9. 用户故事

| 用户 | 场景 | 期望结果 |
| --- | --- | --- |
| 学生 | “我的老师是谁” | 返回该学生相关课程和任课教师 |
| 学生 | “我想请假，明天上午病假，发烧” | 助手补齐请假信息，生成待确认申请 |
| 学生 | “我最近学习压力大” | 先共情，再给出专业、温和、可执行建议 |
| 教师 | “查询我今天的课表” | 返回本人授课安排 |
| 班主任 | “查看本班请假和考勤异常” | 返回本班范围内数据 |
| 管理员 | “查询所有学生” | 返回分页学生信息，并支持继续显示全部/筛选 |
| 管理员 | “把吴磊岗位改为教师” | 识别教师对象，展示修改前后，确认后执行 |
| 任意用户 | “十常侍是谁” | 从知识库检索并用现代白话回答，附简短出处 |

## 10. 权限与安全需求

- AI 不能直接执行 SQL。
- 所有系统操作必须通过后端注册工具。
- 工具执行前必须校验 RBAC 权限和数据范围。
- 学生只能访问自己的成绩、考勤、课表、请假等数据。
- 教师只能访问自己授课、管理班级或院系范围内的数据。
- 管理员拥有全局数据管理能力。
- 新增、修改、删除、批量发送邮件等操作必须经过确认。
- 情绪高风险提醒只做关怀和转介，不做诊断。
