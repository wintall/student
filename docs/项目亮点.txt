核心亮点
不是普通学生管理系统，而是校园综合管理平台
覆盖学生、教师、院系、班级、课程、成绩、考试、课表、请假、考勤、公告、邮件、数据导出、权限管理等模块，业务链条比较完整。

足球智能助手是最大创新点
用户不只是点菜单，而是可以用自然语言完成查询、操作、问答、发邮件、查路线、处理文档、分析代码等任务，更像一个校园智能入口。

主 Agent + 子 Agent 架构
助手不是简单关键词匹配，而是通过主 Agent 判断意图，再分发给教务、RAG、学习辅导、情绪陪伴、地图生活、GitHub、代码体检等子 Agent，后续扩展模块比较方便。

自然语言操作教务数据
管理员可以说“查询所有学生”“张芳性别改为男”“给某个学生发邮件”，系统会识别意图、补齐参数、调用已有业务接口，并对敏感操作做二次确认。

权限控制比较完整
admin、教师、学生、班主任、院系主任等角色权限不同。AI 助手也不是绕过权限，而是在用户权限范围内调用系统能力。

RAG 综合知识库
支持上传 txt、md、pdf、本地路径和长文本，自动解析、切片、向量化，结合 Milvus 做知识库问答。还提供 TopK、切片策略、检索方式、召回率、准确率、F1 等指标展示。

文档处理能力
支持文件总结、图片文字识别 OCR、英汉互译等，体现 AI 在办公场景中的应用。

情绪陪伴与人在回路
学生表达明显负面情绪时，系统可以给出心理支持建议，并在高风险情况下提醒 admin 或相关老师，体现“AI 辅助但人负责”的设计。

路线生活服务
可以查询附近吃喝玩乐、规划路线，校园助手不只服务教务，也服务学生日常生活。

代码体检模块
   用户给出项目路径后，系统可以分析项目结构、技术栈、代码质量、潜在风险和优化建议，并生成报告文件，适合展示 AI 辅助开发能力。

答辩时的一句话
这个系统的最大亮点是：它把传统教务管理系统升级成了一个带 AI Agent 能力的校园智能平台，既能管理真实业务数据，又能通过自然语言、RAG 知识库、文档处理、情绪陪伴、地图服务和代码分析等能力，为不同角色提供智能化服务。

智能模块	用到的 AI / 技术知识
自然语言教务操作	意图识别、槽位抽取、多轮对话、工具调用、权限约束、Human-in-the-loop 二次确认
RAG 知识问答	文档解析、文本切片、Embedding 向量化、Milvus 向量检索、关键词检索、混合召回、TopK、召回率/准确率/F1 评估
学习辅导 / 诗词鉴赏	大语言模型生成、教学式提示词、结构化讲解、知识点拆解、诗歌意象/情感/手法分析
AI 知识问答	大语言模型问答、技术知识提示词、编程知识结构化解释
搜索引擎模块	Web Search、搜索结果摘要、时效性信息增强、来源过滤
文档处理	OCR 图片文字识别、文本抽取、摘要生成、英汉翻译、文档理解
情绪陪伴	情绪识别、心理支持对话、风险等级判断、危机提示、人在回路通知
路线生活	地理编码、POI 搜索、路径规划、地图 API 调用、生活服务推荐
数据分析	数据统计、指标聚合、自然语言解释数据、可视化思维
世界杯问答	主题问答、搜索增强、体育知识查询、时效性纠错
GitHub 助手	GitHub API 调用、仓库解析、Issue 创建、公开代码下载、工具调用
代码体检	静态代码分析、目录结构识别、复杂度/风险评估、项目总结、报告生成
足球助手 UI	Three.js / CSS 动画、交互式悬浮入口、模块化能力切换

你答辩时可以总结成四类：
Agent 类 AI：自然语言教务操作、GitHub、代码体检
核心是“识别意图，然后调用工具完成任务”。

RAG 类 AI：综合知识库、AI 知识问答、部分学习辅导
核心是“先检索资料，再基于资料回答”。

多模态/文档 AI：OCR、翻译、文件总结
核心是“把图片或文档转成可理解文本，再处理”。

场景智能服务：情绪陪伴、路线生活、数据分析
核心是“结合具体校园场景做智能判断和服务”。
