# 二阶段接口设计文档：校园 AI 助手中心

版本：v0.1  
日期：2026-06-26  
状态：开发前草案，接口会随开发补充

## 1. 统一规范

- 基础路径：`/api/v1`
- 鉴权方式：沿用 JWT Bearer Token
- 响应格式：沿用 `{code, message, data}`
- 权限控制：后端 `require_permission` + 工具内部数据范围校验

## 2. 校园助手接口

### 2.1 发送消息

```http
POST /campus-agent/chat
```

请求：

```json
{
  "session_id": "optional-session-id",
  "mode": "auto",
  "message": "我想查一下我的成绩"
}
```

响应：

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "session_id": "session-id",
    "reply": "这是你最近的成绩...",
    "mode": "academic_tools",
    "intent": "query_my_scores",
    "tool_calls": [
      {
        "tool": "get_my_scores",
        "status": "success",
        "duration_ms": 120
      }
    ],
    "references": [],
    "suggested_questions": [
      "帮我分析一下成绩变化",
      "帮我制定复习计划"
    ]
  }
}
```

### 2.2 创建会话

```http
POST /campus-agent/sessions
```

响应：

```json
{
  "session_id": "xxx",
  "title": "新会话",
  "mode": "auto"
}
```

### 2.3 会话列表

```http
GET /campus-agent/sessions
```

### 2.4 会话消息

```http
GET /campus-agent/sessions/{session_id}/messages
```

### 2.5 删除会话

```http
DELETE /campus-agent/sessions/{session_id}
```

### 2.6 用户反馈

```http
POST /campus-agent/messages/{message_id}/feedback
```

请求：

```json
{
  "rating": 1,
  "comment": "回答有帮助"
}
```

## 4. 工具调用接口设计

工具不建议全部直接暴露给前端。前端主要调用 `/campus-agent/chat`，由 Agent 内部调用工具。

内部工具建议：

| 工具 | 用途 | 权限 |
| --- | --- | --- |
| get_my_scores | 查询本人成绩 | 登录用户 |
| get_student_scores | 查询学生成绩 | 教师范围 / 管理员 |
| get_my_schedule | 查询本人课表 | 登录用户 |
| get_recent_announcements | 查询公告 | 登录用户 |
| get_my_leave_requests | 查询本人请假 | 登录用户 |
| get_pending_leave_reviews | 待审批请假 | 班主任 / 院系主任 / 管理员 |
| get_my_attendance | 查询本人考勤 | 登录用户 |
| get_student_attendance | 查询学生考勤 | 班主任 / 院系主任 / 管理员 |
| get_teacher_attendance | 查询教师考勤 | 院系主任 / 管理员 |
| data_health_summary | 数据体检摘要 | 授权角色 |
| search_rag_knowledge | RAG 知识问答 | 登录用户 |
| search_ai_knowledge | AI 知识问答 | 登录用户 |
| search_worldcup_knowledge | 世界杯问答 | 登录用户 |

## 5. 请假考勤接口

建议将前端模块命名为“请假考勤”，后端可以保留现有 `/leave` 接口，再新增 `/attendance`。

### 5.1 我的考勤

```http
GET /attendance/my
```

查询参数：

| 参数 | 说明 |
| --- | --- |
| start_date | 开始日期 |
| end_date | 结束日期 |
| status | 考勤状态 |
| page | 页码 |
| page_size | 每页条数 |

### 5.2 学生考勤列表

```http
GET /attendance/students
```

权限：

- 班主任：本班学生。
- 院系主任：本院系学生。
- admin：全部学生。

查询参数：

| 参数 | 说明 |
| --- | --- |
| clazz_id | 班级 |
| department_id | 院系 |
| student_id | 学生 |
| start_date | 开始日期 |
| end_date | 结束日期 |
| status | 状态 |

### 5.3 教职工考勤列表

```http
GET /attendance/teachers
```

权限建议：

- 院系主任：本院系教职工。
- admin：全部教职工。

### 5.4 创建考勤记录

```http
POST /attendance
```

请求：

```json
{
  "person_type": "student",
  "user_id": 12,
  "attendance_date": "2026-06-26",
  "period_type": "day",
  "status": "normal",
  "remark": "正常出勤"
}
```

### 5.5 更新考勤记录

```http
PUT /attendance/{id}
```

### 5.6 删除考勤记录

```http
DELETE /attendance/{id}
```

### 5.7 请假同步考勤

```http
POST /attendance/sync-leave/{leave_request_id}
```

说明：请假审批通过后由服务层自动调用，第一版也可做成内部函数，不一定暴露给前端。
