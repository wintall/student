# 二阶段接口设计文档：校园 AI 助手中心

版本：v1.0  
日期：2026-06-30  
状态：二阶段交付版

## 1. 统一规范

- 基础路径：`/api/v1`
- 鉴权方式：沿用 JWT Bearer Token
- 响应格式：沿用 `{code, message, data}`
- 权限控制：后端 `require_permission` + 工具内部数据范围校验

## 2. 校园助手接口

### 2.1 发送消息

```http
POST /campus-agent/chat
```

请求：

```json
{
  "session_id": "optional-session-id",
  "mode": "auto",
  "message": "我想查一下我的成绩"
}
```

响应：

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "session_id": "session-id",
    "reply": "这是你最近的成绩...",
    "mode": "academic_tools",
    "intent": "query_my_scores",
    "tool_calls": [
      {
        "tool": "get_my_scores",
        "status": "success",
        "duration_ms": 120
      }
    ],
    "references": [],
    "suggested_questions": [
      "帮我分析一下成绩变化",
      "帮我制定复习计划"
    ]
  }
}
```

### 2.2 创建会话

```http
POST /campus-agent/sessions
```

响应：

```json
{
  "session_id": "xxx",
  "title": "新会话",
  "mode": "auto"
}
```

### 2.3 会话列表

```http
GET /campus-agent/sessions
```

### 2.4 会话消息

```http
GET /campus-agent/sessions/{session_id}/messages
```

### 2.5 删除会话

```http
DELETE /campus-agent/sessions/{session_id}
```

### 2.6 用户反馈

```http
POST /campus-agent/messages/{message_id}/feedback
```

请求：

```json
{
  "rating": 1,
  "comment": "回答有帮助"
}
```

## 4. 工具调用接口设计

工具不建议全部直接暴露给前端。前端主要调用 `/campus-agent/chat`，由 Agent 内部调用工具。

内部工具建议：

| 工具 | 用途 | 权限 |
| --- | --- | --- |
| get_my_scores | 查询本人成绩 | 登录用户 |
| get_student_scores | 查询学生成绩 | 教师范围 / 管理员 |
| get_my_schedule | 查询本人课表 | 登录用户 |
| get_recent_announcements | 查询公告 | 登录用户 |
| get_my_leave_requests | 查询本人请假 | 登录用户 |
| get_pending_leave_reviews | 待审批请假 | 班主任 / 院系主任 / 管理员 |
| get_my_attendance | 查询本人考勤 | 登录用户 |
| get_student_attendance | 查询学生考勤 | 班主任 / 院系主任 / 管理员 |
| get_teacher_attendance | 查询教师考勤 | 院系主任 / 管理员 |
| data_health_summary | 数据体检摘要 | 授权角色 |
| search_rag_knowledge | RAG 知识问答 | 登录用户 |
| search_ai_knowledge | AI 知识问答 | 登录用户 |
| search_worldcup_knowledge | 世界杯问答 | 登录用户 |

## 5. 请假考勤接口

| start_date | 开始日期 |
| end_date | 结束日期 |

### 5.3 教职工考勤列表

```http
GET /attendance/teachers
```

- admin：全部教职工。


```json
  "attendance_date": "2026-06-26",
  "period_type": "day",
  "status": "normal",
  "remark": "正常出勤"
```


### 5.6 删除考勤记录


### 5.7 请假同步考勤


### 9.3 综合知识库 RAG

基础路径：`/api/v1/rag/knowledge`

| 方法 | 路径 | 用途 | 权限 |
| --- | --- | --- | --- |
| POST | `/bases` | 创建知识库 | 登录用户；公共库需管理员 |
| GET | `/bases` | 查询可访问知识库 | 登录用户 |
| PUT | `/bases/{kb_id}` | 修改知识库 | 所有人或管理员 |
| DELETE | `/bases/{kb_id}` | 删除知识库 | 所有人或管理员 |
| GET | `/bases/{kb_id}/documents` | 查询知识库文档 | 可读该知识库 |
| POST | `/documents/text` | 导入文本 | 可管理该知识库 |
| POST | `/documents/path` | 从白名单本地路径导入 | 可管理该知识库 |
| POST | `/documents/upload` | 上传文件导入 | 可管理该知识库 |
| DELETE | `/documents/{document_id}` | 删除文档 | 所有人或管理员 |
| POST | `/search` | 检索知识库片段 | 可读该知识库 |
| POST | `/ask` | 知识库问答 | 可读该知识库 |
| GET | `/health` | 知识库健康信息 | 登录用户 |

`POST /rag/knowledge/bases`：

```json
{
  "name": "三国演义知识库",
  "description": "文学问答资料",
  "scope_type": "personal",
  "scope_id": null
```

```json
{
  "kb_id": 1,
  "title": "三国演义",
  "text": "待导入的长文本..."
}
```

```json

  "question": "十常侍是谁",

  "min_score": 0.0
```

## 10. 接口安全约束

- 所有接口默认要求登录，除登录/公开资源外不允许匿名访问。
- 自然语言工具调用必须经过后端权限校验，不能只依赖前端菜单。
- RAG 公共知识库只有管理员可创建，个人知识库仅所有人和管理员可管理。
- 考勤详情查询除权限码外，还必须做数据范围校验。
- 写操作、删除操作、批量操作必须走确认机制。
